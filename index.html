mscoco 분류해보기

Inception_v3에 사용해볼 데이터 모으기

Inception_v3 등 다양한 오픈소스를 재학습해 사용해보기 위해선 분류된 이미지 데이터가 필요하다.
이미지 데이터를 모으기 위해 MSCOCO에서 제공되는 train, val, instances 를 이용해서 분류시켜보았다.

  • MSCOCO: http://cocodataset.org/#home

val2017 데이터와 instances_val2017를 사용하였고 결과는 classification에 저장되도록 했다.

실행 Run! 5000개의 이미지가 80개의 카테고리로 분류가 된다.

분류 후 classification 폴더 내 결과

개선해야 할 점 & 후기

json형식으로된 instances를 사용해 읽어오는 것이라 어렵진 않았지만 조금 복잡하긴 했다. 결과를 빨리 내야해서 코드 유연성이나 효율면에서 많이 부족한 코드다. 진행도 같은 것들을 추가하여 진행률을 알 수 있도록 하면 좋을 것 같다.

1. 작품소개

ㅇ 기획의도

  • 태그에 기반한 추천 방식은 변화가 빠른 기사 추천의 어려움을 보완하는 동시에 사용자-기사 관계의 특징이 데이터 희소성 문제를 해결해 준다.
  • 텍스트뿐만이 아닌 기사의 사진에서도 태그를 추출함으로써 한국어 형태소 분류 시 발생하는 오류로 인한 문제를 완화할 수 있다.
  • 기사 외에도 SNS와 같은 텍스트와 이미지를 동시에 갖는 도메인에 확장될 수 있다.

4. 후기

학교 졸업 프로젝트로 개발했던 쿼드코어 뉴스. 나는 태그 추출과 기사 추천 부분에 참여했다.
태그 추출에 있어 기존에 제공되는 Python 라이브러리인 Konlpy를 사용하여 명사를 추출하였고 이를 이용해 TF-IDF방식을 사용하여 기사 내용 기반 텍스트 태그를 추출하였다.
이미지 태그는 구글의 Inception_v3모델을 사용하였다. 직접 모은 이미지 데이터들을 이용해 재 학습시키는 방식으로 태그를 추출하였다. 태그가 달린 이미지를 모으는데 있어 MSCOCO 데이터를 사용해보기도 했다. 긴 시간동안 다양한 소스들을 사용해보았지만 그만큼 완성도에 있어 아쉬운 점도 크다. 추천 방식을 정하는 것에 대한 어려움이 많았고 그 때문에 시간이 오래 걸리기도 했다.

duxby (빅스비 챕슐 챌린지 2019)

duxby

삼성 빅스비 스튜디오를 사용한 빅스비 캡슐 챌린지

빅스비 챕슐 챌린지 2019 입선

주요 기능

  1. findFestByArea(sido,gugun) 시,군
    • 시,군 을 입력하여 그 지역에서 진행하고 있는 행사들을 제공
  2. findFestByRealm(sido,gugun,realmName) (realmName: 연극,뮤지컬,콘서트,음악,무용,미술…)
    • 종류 를 입력하여 그 종류에 해당하는 행사들을 제공
  3. findFestByThisMonth(이번달)
    • 이번 달 을 입력하여 이번 달에 진행되고 있는 행사들을 제공
  4. findFestByThisWeek(이번주)
    • 이번 주 를 입력하여 이번 주에 진행되고 있는 행사들을 제공
  5. findFestBySelectMonth(월 선택)
    • 월 을입력하여 그 월에 진행되고 있는 행사들을 제공

구동 화면

  • 발화: 12월 행사 보여줘.

  • 발화: 서울 중구의 축제 보여줘

  • 발화: 연극 뭐 볼까?

  • 발화: 이번 주 공연 보여줘(해당하는 공연이 없을 시 화면)

아쉬운 점

  • 적은 발화 목록
  • 공공 데이터 API 오류
  • 빅스비 스튜디오 활용 부족

후기

나의 첫 자바스크립트 프로젝트였던 빅스비 캡슐 챌린지. 학기 중에 도전했었던 공모전으로 결과물에서 최대한 정확하게 정보를 전달 하도록 노력을 했다. 타 팀에 비해 자바스크립트와 빅스비 스튜디오에 대한 이해는 부족할 수 있었지만 직접 빅스비를 사용하는 유저였고 빅스비와 연동되는 앱이 적다는 점을 노려 공공 API(공연전시정보조회서비스)를 사용하여 공연 정보를 전달하고 공연사이트로 연결 해주는 실용적인 주제로 접근했다.
이번 주, 이번 달, 월 별 공연 발화를 담당했었는데 시간이 부족해 발화가 너무 적었고 빅스비 개발자도구를 잘 사용하지 못해서 아쉬웠다.

시연동영상

1. 작품소개

ㅇ 기획의도

  • 엔도르핀 분비 증가 등 웃음의 긍정적인 효과에 집중하여 우울증. 불안 장애와 같은 정신 질환을 비교적 많이 겪는 현대인들을 지금보다 더 많이 웃게 만들어 보다 나은 삶을 살아갈 수 있도록 돕고자 함. 또한 웃는 것에 대한 거부감을 줄이고 인상이 개선될 수 있도록 도움.

  • 기존의 인공지능 모델이 동양 여성에게 있어 인식률이 떨어지는 디지털 편향성을 해결하기 위한 해결책을 제시하고자 함.

  • 비전문가들도 인공지능 서비스를 친숙하고 흥미롭게 경험할 수 있도록 함.

2. 구성도

  1. 작품 구성도

  1. 데이터 수집 시스템 구성도

  1. 학습 시스템 구성도

3. 기능

1) 웃어서 잠금 해제 : 웃는 경우에만 어플 잠금을 해제하는 기능

2) 스마일 챌린지 : 웃는 횟수 등으로 구성된 데일리 미션

3) 웃음 일기 : 잠금 해제 시 저장된 사용자의 웃는 사진들을 일기 형식으로 수치와 함께 보여줌

4) 알람 : 위치기반, 사용자가 원하는 시간, 랜덤 알람

4. 개선해야 할 점

1) 사진 저장 시 회전 문제

2) 웃음 감정 분석 속도 개선

5. 후기

안드로이드 수강 후 처음으로 팀원과 함께 공모전 제출을 위해 오픈 소스들을 사용하여 만들었던 작품이다. 운 좋게 첫 공모전을 입선으로 마무리 할 수 있었다.
처음엔 모든 감정을 다 나타낼수 있도록 하는 앱으로 가려 했으나 데이터 문제 때문에 웃는 것과 무표정 2가지로 분류하였다.
MicroSoft FACE API를 통해 데이터를 수집하였고, Google Cloud를 사용했다.
Google Cloud에는 첫 사용자에게 무료 토큰 300달러가 주어져서 Google Cloud를 사용하여 사진 데이터를 넣고 학습 시켜 배포 하도록 했다.

Hexo

Hexo를 이용하여 블로그 시작하기

마크 다운으로 적어보는 블로그 첫 글

hexo vs jekyll = node.js vs Ruby

몇년 전 동아리 활동을 통해 Ruby on ralis 를 배웠던 적이 있어 익숙한 것을 고르자면 루비지만
요새 관심을 두고 있는건 node.js와 React 쪽이라 hexo를 선택 했다.

검색해보면 hexo로 블로그 생성부터 테마 생성 등등 다양한 정보를 얻을 수 있었다.
처음 시작은 늘 그렇듯 install 부터다.

  1. npm install hexo-cli -g

  2. cd로 원하는 폴더로 이동 후

  3. hexo init (원하는 폴더명)

  4. server 실행 해보기! - > hexo server

중간에 났던 오류들

1. _config.yml에 설정 중 오류

deploy:
type: git
repo: “깃 주소”
branch: master

ERROR Deployer not found: git
이번 오류는 단순하다. hexo 폴더에 깃 설치를 하지 않았다..^^;;

npm install hexo-deployer-git –save 로 해결!

2. 테마 적용 오류

사실 처음엔 정리되어 있는 테마를 설정하려다 새 테마들이 기본 적으로 가지고 있는 메뉴 들과 파일이 많아
일단 기본 테마인 landscape에 어느 정도 적응을 해보려 한다. 전에 테마 적용 중 _config.yml에 theme에 제대로 입력을 했지만 .js .css가 로컬에서 적용되지않았다.

타 블로그 댓글에서 알려주신 방법을 통해 해결했다.
-> cd 로 적용 할 테마 폴더 까지 이동 후

서버 재 실행을 하면 이쁘게 적용이 잘 되어있다.
